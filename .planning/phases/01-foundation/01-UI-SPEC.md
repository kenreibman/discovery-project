## Page Contracts

### Login Page (`/login`)

**Layout:** Full viewport height. Cream background (#F5F3EE). Single white card centered both horizontally and vertically. Card max-width 400px, padding 32px.

**Card contents (top to bottom):**
1. App name "Discovery Drafter" in 16px weight 600 text-secondary color, centered. No logo.
2. 24px vertical space
3. Email input with label "Email" (14px, weight 400)
4. 16px vertical space
5. Password input with label "Password" (14px, weight 400), type="password"
6. 8px vertical space (reserved for inline error)
7. Inline error text area: 14px, #DC2626, hidden when no error. Shows "Invalid email or password" on failed login. No toast.
8. 24px vertical space
9. Submit button "Sign in" full width, accent background (#C8653A), white text, 16px weight 600, 44px height, 8px border-radius

**States:**
- Default: Fields empty, button enabled
- Loading: Button shows spinner icon (lucide `Loader2` with spin animation), text changes to "Signing in...", button disabled
- Error: Inline red text appears below password field. Fields retain entered values. Button re-enabled.
- Success: Redirect to `/` (dashboard). No success toast.

**Keyboard:** Enter key submits form. Tab order: email, password, submit.

### Dashboard (`/`)

**Layout:** Full viewport. Sidebar 280px wide, fixed left. Main content area fills remaining width, centered content with max-width 640px.

**Sidebar (top to bottom):**
1. App name "Discovery Drafter" in 16px weight 600, padding 24px horizontal, 24px top
2. 16px separator
3. Case list area (scrollable, flex-grow)
   - Each entry: case name (16px, weight 400, truncated with ellipsis) + date below (14px, text-secondary). Padding 12px horizontal, 8px vertical. Left border 3px transparent (active = accent).
   - Empty state: "No cases yet" centered in case list area, 14px, text-secondary (#78736A)
4. Bottom section (sticky):
   - 16px separator
   - Logout row: lucide `LogOut` icon (16px) + "Log out" text (14px, weight 400, text-secondary). Ghost button style. Padding 24px horizontal, 16px vertical.

**Main content area:**
1. 64px top padding
2. Greeting: "Good morning, Jessica" (24px, weight 600, text-primary). Time-of-day logic: 5-11 = morning, 12-16 = afternoon, 17+ = evening.
3. 8px vertical space
4. Subtitle: "Upload a complaint and discovery request to get started." (16px, weight 400, text-secondary #78736A)
5. 32px vertical space
6. Drop zone: Dashed border (2px, #E5E0D8), border-radius 12px, padding 48px vertical. Centered content:
   - lucide `Upload` icon, 32px, text-secondary
   - 16px vertical space
   - "Drop files here or click to browse" (16px, weight 400, text-secondary)
   - 8px vertical space
   - "PDF files up to 20MB" (14px, weight 400, text-secondary, lower opacity)

**Drop zone states (Phase 1 shell only -- no actual upload processing):**
- Default: Dashed border #E5E0D8, icon and text in text-secondary
- Hover (mouse): Border color transitions to #C8653A, background shifts to #C8653A08 (accent at 3% opacity)
- Drag-over: Same as hover but with border solid instead of dashed
- Click: Opens file picker (Phase 2 will wire the handler; Phase 1 renders the UI only)

### Redirect Behavior

- Unauthenticated visit to `/` redirects to `/login` (silent, no error message) -- per D-12
- Authenticated visit to `/login` redirects to `/` (no flash of login page)
- Session expiry during use: silent redirect to `/login`, after re-login return to `/` -- per D-12

## Interaction Contracts

| Interaction | Behavior |
|------------|----------|
| Login form submit | POST credentials, show loading spinner on button, disable button. On 401: show inline error, re-enable button. On 200: redirect to `/`. |
| Sidebar case click | Navigate to case detail page (shell only in Phase 1 -- route exists but content is placeholder). Active state: accent left border + darker background. |
| Sidebar scroll | Case list scrolls independently. Sticky app name at top, sticky logout at bottom. |
| Drop zone click | Opens native file picker (accept=".pdf"). No upload processing in Phase 1. |
| Drop zone drag | Visual feedback on dragenter/dragover (border and background change). Revert on dragleave. No file processing in Phase 1. |
| Logout click | End session, redirect to `/login`. No confirmation dialog. |
| Responsive | Not required for v1 (Jessica uses desktop). Sidebar collapses are deferred. Minimum supported width: 1024px. |

## Accessibility Baseline

| Concern | Implementation |
|---------|---------------|
| Focus indicators | 2px ring in accent color (#C8653A) with 2px offset on all focusable elements |
| Color contrast | Text primary (#1A1714) on dominant (#F5F3EE) = ratio > 12:1. Text secondary (#78736A) on dominant = ratio > 4.5:1. White on accent button = ratio > 4.5:1. |
| Error association | Login error text linked to password field via `aria-describedby` |
| Form labels | All inputs have visible `<label>` elements with `htmlFor` binding |
| Keyboard nav | Full tab navigation. Enter submits login form. No keyboard traps. |
| Motion | Loading spinner respects `prefers-reduced-motion` (stops animation) |
