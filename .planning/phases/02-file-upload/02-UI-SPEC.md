# Phase 2 — UI Design Contract

## Page Contracts

### Dashboard (`/dashboard`) — Updated

**Layout:** Carries forward from Phase 1. Sidebar 280px, main content max-width 640px centered.

**Main content area (top to bottom):**
1. 64px top padding (existing)
2. Greeting (existing, unchanged)
3. 8px vertical space
4. Subtitle: Changes to "Upload discovery request PDFs to start a new case." (16px, weight 400, text-secondary)
5. 32px vertical space
6. Upload zone (extended from `UploadZoneShell` -- see Upload Zone section below)
7. 32px vertical space
8. File list (appears inline below drop zone when files are selected -- see File List section)
9. 32px vertical space
10. "Recent Cases" section heading: "Recent Cases" (16px, weight 600, text-primary)
11. 16px vertical space
12. Case list (see Case List section)

### Upload Zone (extended from UploadZoneShell)

**Visual:** Carries forward existing dashed border zone exactly. `multiple` attribute added to file input. Accept `.pdf` only.

**New behavior:**
- On file selection (drop or pick), files appear in the File List below the zone
- Drop zone remains visible and interactive after file selection (allows adding more files)
- 20MB per-file size limit enforced client-side. Files exceeding limit show inline error in file list, not a toast.

### File List (inline below drop zone)

**Appears:** Only when one or more files have been selected. No page navigation.

**Layout:** Vertical stack of file rows. Each row is a horizontal flex container with 16px padding vertical, separated by 1px #E5E0D8 borders.

**Each file row contains (left to right):**
1. File icon: lucide `FileText`, 20px, text-secondary (#78736A)
2. 8px horizontal gap
3. File info column (flex-grow):
   - Filename (16px, weight 400, text-primary, truncated with ellipsis at 300px max-width)
   - Below filename: file size in human-readable format (14px, weight 400, text-secondary). Example: "2.4 MB"
4. 8px horizontal gap
5. Type badge area:
   - During classification: skeleton shimmer, 80px wide, 24px tall, border-radius 4px
   - After classification: badge showing "Complaint" or "Discovery Request" (14px, weight 400). Background: `--secondary` (#F0EDE6), text: `--foreground`. Click badge to open type override select dropdown.
6. 8px horizontal gap
7. Status area (right-aligned, 120px min-width):
   - Uploading: progress bar (4px tall, accent fill, 120px wide) with percentage below (14px, text-secondary)
   - Classifying: "Classifying..." text (14px, text-secondary) with lucide `Loader2` spinning icon (14px)
   - Done: lucide `Check` icon (16px, #16A34A green)
   - Error: lucide `AlertCircle` icon (16px, destructive #DC2626) with error text below (14px, destructive)

**File row states:**
- Uploading: Progress bar animates. No actions available.
- Classifying: Shimmer on badge area. No actions available.
- Complete: Badge is clickable for type override. Remove button appears on hover.
- Error: Error icon and message. Retry button (14px, accent color text, underline on hover).

### "Create Case" Button

**Appears:** Below the file list, after all files reach "done" or "error" status. Right-aligned.

**Visual:** Primary button (accent background #C8653A, white text "Create Case", 16px weight 600, 44px height, 8px border-radius). Disabled state while any file is still uploading or classifying.

**Behavior:** Creates case record in database, links uploaded files, navigates to `/case/[id]`.

**Loading state:** Button text changes to "Creating..." with lucide `Loader2` spin animation. Button disabled.

### Case List (dashboard, below upload zone)

**Layout:** Vertical stack of case rows. Each row is a clickable link to `/case/[id]`.

**Each case row contains:**
- Left: Case name (16px, weight 400, text-primary). If name is null, show "Untitled Case" in text-secondary italic.
- Right: Document count badge ("2 docs", 14px, text-secondary) + created date (14px, text-secondary, relative format: "Today", "Yesterday", "Mar 28").
- Row padding: 12px vertical, 0 horizontal. Bottom border 1px #E5E0D8.
- Hover: Background transitions to #F0EDE6 (--secondary).
- Active: No special state on dashboard list (active state is on sidebar only).

**Empty state:** Hidden entirely when no cases exist (the existing "No cases yet" in sidebar covers this).

**Sort order:** Most recent first (by `createdAt` descending).

**Limit:** Show last 10 cases. No pagination in Phase 2.

### Case Detail Page (`/case/[id]`)

**Layout:** Same protected layout with sidebar. Main content max-width 640px centered.

**Content (top to bottom):**
1. 48px top padding
2. Back link: lucide `ArrowLeft` (16px) + "Back to Dashboard" (14px, text-secondary). Hover: text-primary. Links to `/dashboard`.
3. 24px vertical space
4. Case name: Editable inline heading (24px, weight 600). Click to edit -- transforms to an input field with the same styling. Press Enter or blur to save. Placeholder: "Untitled Case" in text-secondary.
5. 8px vertical space
6. Case metadata: "Created [date]" (14px, text-secondary). Absolute date format: "April 1, 2026".
7. 32px vertical space
8. "Documents" section heading (16px, weight 600) with "Add Documents" button (outline variant, 14px, lucide `Plus` icon 14px) right-aligned on the same line.
9. 16px vertical space
10. Document list: Same visual structure as the file list rows, but without progress bars. Each row shows:
    - File icon (lucide `FileText`, 20px, text-secondary)
    - Filename (16px, weight 400, text-primary)
    - Type badge ("Complaint" or "Discovery Request", clickable to override)
    - Uploaded date (14px, text-secondary, relative format)
    - Action menu (lucide `MoreVertical`, 16px, text-secondary): dropdown with "Remove" option
11. 48px vertical space
12. Danger zone: "Delete Case" button (destructive ghost variant, 14px, lucide `Trash2` icon 14px + "Delete Case" text, text destructive #DC2626). Left-aligned.

**"Add Documents" flow:** Opens the same file picker (accept `.pdf`, multiple). New files upload to Vercel Blob and appear in the document list with progress indicators. Same upload/classify flow as dashboard.

### Sidebar Updates

**Case list in sidebar:** Replace "No cases yet" with actual case entries when cases exist.

**Each sidebar case entry:**
- Case name (16px, weight 400, truncated with ellipsis). "Untitled Case" in text-secondary italic if name is null.
- Document count below (14px, text-secondary): "2 documents"
- Padding: 12px horizontal, 8px vertical
- Hover: background #F0EDE6
- Active (current page matches `/case/[id]`): background #EDE8DF, left border 3px #C8653A
- Click: navigate to `/case/[id]`

**Sidebar sort:** Most recent first.

## Interaction Contracts

| Interaction | Behavior |
|------------|----------|
| File drop/select | Files appear in inline file list below drop zone. Each file begins uploading to Vercel Blob immediately via client-side upload. After upload completes, classification request fires automatically. |
| Upload progress | Progress bar fills from 0-100% with percentage text. Smooth animation (CSS transition, 200ms). |
| Classification | Skeleton shimmer on badge area during classification (1-3 seconds). Badge appears with detected type on completion. |
| Type override | Click badge to open select dropdown with options: "Complaint", "Discovery Request". Selection updates the document record immediately (optimistic UI, server action in background). |
| Create Case click | Validates at least one file is in "done" state. Creates case record, links documents, navigates to `/case/[id]`. Shows toast on error. |
| Case row click (dashboard) | Navigate to `/case/[id]`. |
| Case row click (sidebar) | Navigate to `/case/[id]`. Active state updates immediately. |
| Case name edit | Click case name heading on case detail page. Transforms to input. Enter or blur saves. Escape reverts. Server action updates database. |
| Add Documents click | Opens native file picker. Selected files upload and classify using same flow as dashboard. New documents append to list. |
| Remove document | Click `MoreVertical` to open dropdown, click "Remove". No confirmation dialog (non-destructive at document level, files can be re-uploaded). Document row animates out (opacity 0, height 0, 200ms). Server action deletes document record and Vercel Blob file. |
| Delete Case | Click "Delete Case" button. AlertDialog opens with confirmation copy. "Delete Case" button in dialog is destructive variant. On confirm: deletes case and all documents, navigates to `/dashboard`, shows success toast "Case deleted". |
| Sidebar case list | Loads case list on initial page load. Updates when cases are created or deleted (revalidation via server actions). |
| Responsive | Not required for v1. Minimum supported width: 1024px. |

## Accessibility

| Concern | Implementation |
|---------|---------------|
| Focus indicators | 2px ring in `--ring` (#C8653A40) with 2px offset on all interactive elements (carries forward) |
| Drop zone keyboard | Space/Enter opens file picker (existing). `role="button"` and `tabIndex={0}` (existing). |
| Progress bars | `role="progressbar"` with `aria-valuenow`, `aria-valuemin="0"`, `aria-valuemax="100"`, `aria-label="Uploading [filename]"` |
| File list | Use `role="list"` and `role="listitem"` for file rows. Status changes announced via `aria-live="polite"` region. |
| Type override select | Accessible label "File type for [filename]". Keyboard navigable via Radix Select. |
| Delete confirmation | AlertDialog traps focus. Escape closes. Cancel button focused by default. |
| Case name edit | Input has `aria-label="Case name"`. Screen readers announce "Editing case name" on activation. |
| Color contrast | All text/background combinations maintain WCAG AA (4.5:1 minimum). Green check (#16A34A) on white (#FFFFFF) = 4.5:1. |
