2. Create `src/components/upload-zone.tsx` -- full upload zone replacing UploadZoneShell ("use client"):

This component manages the entire upload flow state. It renders the drop zone visuals (carry forward from `UploadZoneShell`) plus the file list and Create Case button.

State management:
```typescript
const [files, setFiles] = useState<FileUploadState[]>([]);

```

Carry forward from UploadZoneShell:
- Same drag/drop handlers (handleDragEnter, handleDragOver, handleDragLeave, handleDrop)
- Same visual styling (dashed border, hover/active states with #C8653A)
- Same keyboard handling (Space/Enter opens file picker)
- `role="button"`, `tabIndex={0}`

Key changes from shell:
- File input gets `multiple` attribute per D-01
- `accept=".pdf,application/pdf"` on file input
- `handleDrop` and `onChange` call a shared `handleFiles(fileList: FileList)` function

`handleFiles` function:
1. Convert FileList to array, validate each with `validateFile()` from `src/lib/upload.ts`
2. Invalid files: add to state with status "error" and the validation error message
3. Valid files: add to state with status "uploading", progress 0
4. For each valid file, call `uploadSingleFile(file)` in parallel (do NOT await sequentially)

`uploadSingleFile` function:
1. Import `upload` from `@vercel/blob/client`
2. Call `upload(file.name, file, { access: "private", handleUploadUrl: "/api/upload", multipart: true, onUploadProgress: ({ percentage }) => { updateFileState(id, { progress: percentage }) } })`
3. On success: update state to status "classifying", set blobUrl
4. Call `classifyDocument(blob.url)` server action (imported from `@/actions/classify`)
5. On classify success: update state to status "done", set type
6. On classify error: update state to status "done", set type to "discovery_request" (fallback). Show toast: "Could not detect file type. Select type manually."
7. On upload error: update state to status "error", set error message "Upload failed. Click to retry."

`handleRetry(id)`: Reset file state to "uploading", progress 0, re-run uploadSingleFile.

`handleRemove(id)`: Remove file from state array (before case creation, no blob cleanup needed since case doesn't exist yet).

`handleTypeChange(id, type)`: Update file state's type field.

Create Case button rendering:
- Show only when `files.length > 0` and every file has status "done" or "error" (i.e., no "uploading" or "classifying" in progress)
- At least one file must be in "done" state (can't create empty case)
- Right-aligned below the file list
- Primary button variant: bg-[#C8653A], text white, "Create Case" text (16px, weight 600, 44px height, rounded-lg)
- Disabled while `isCreating` is true
- Loading state: text "Creating...", lucide `Loader2` spin animation, button disabled
- On click:
  1. Set `isCreating(true)`
  2. Collect all "done" files into `documentData` array: `{ blobUrl, filename: file.name, type, mimeType: file.type }`
  3. Call `createCase(documentData)` server action
  4. On success: `router.push(\`/case/${result.id}\`)` (use `useRouter` from `next/navigation`)
  5. On error: show toast with error message, set `isCreating(false)`

Subtitle text update: The component itself does NOT change the subtitle. That will be in the dashboard page update.

File list rendering:
- Only render when `files.length > 0`
- Wrapper div with `role="list"` per UI-SPEC accessibility
- Map over `files` state, render `<FileRow>` for each
- 32px gap between drop zone and file list (via parent layout)

function formatRelativeDate(date: Date | null): string {
  if (!date) return "";
  const now = new Date();
  const diff = now.getTime() - date.getTime();
  const days = Math.floor(diff / (1000 * 60 * 60 * 24));
  if (days === 0) return "Today";
  if (days === 1) return "Yesterday";
  return date.toLocaleDateString("en-US", { month: "short", day: "numeric" });
}
